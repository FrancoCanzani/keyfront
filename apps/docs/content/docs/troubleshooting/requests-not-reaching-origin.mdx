---
title: Requests are not reaching the origin
description: Diagnose routing, authentication, and origin connection failures.
---

1. Confirm the request uses the gateway hostname assigned to the service.
2. Confirm the API key is active and belongs to that service.
3. Check the returned HTTP status against the [error reference](/gateway/errors).
4. Confirm the origin URL uses HTTPS and is reachable from outside your network.
5. Confirm the origin accepts the configured `X-Gateway-Secret`.

If Keyfront returns an origin error, record the response status, request time,
gateway hostname, and request ID before contacting support. Never include a full
API key in a support request.
